PASSO A PASSO REAL - DOCKER COM SEUS 6 MODELOS
📌 ANTES DE COMEÇAR

REQUISITOS MÍNIMOS:

    Windows 10 ou 11

    Placa de vídeo NVIDIA GTX 1070 (você já tem)

    Driver da NVIDIA atualizado (versão 450.80.02 ou mais nova)

    Pelo menos 20GB de espaço livre em disco

PASSO 1: INSTALAR O DOCKER

    Abra o Google Chrome

    Digite: docker desktop download

    Clique no primeiro link (docker.com/products/docker-desktop)

    Clique no botão azul "Download for Windows"

    Quando baixar, abra o arquivo

    Marque a opção "Use WSL 2 instead of Hyper-V"

    Clique em "OK"

    Quando terminar, clique em "Restart" (vai reiniciar o PC)

APÓS REINICIAR:
9. O Docker vai abrir sozinho - espere a mensagem "Docker Desktop is running"
10. Aceite os termos (clicar em "Accept")
PASSO 2: INSTALAR SUPORTE A GPU NVIDIA

    Abra o Google Chrome

    Digite: nvidia container toolkit install windows

    Entre no site docs.nvidia.com

    Procure o link para Windows

    Baixe o instalador (é um arquivo .exe)

    Execute e instale (next, next, finish)

PARA VERIFICAR SE DEU CERTO:

    Abra o "Prompt de Comando" (tecla Windows + R, digite cmd, Enter)

    Digite: docker run --rm --gpus all nvidia/cuda:11.0-base nvidia-smi

    Se aparecer uma tabela com sua placa de vídeo, deu certo

PASSO 5: INICIAR OS CONTAINERS

    Abra o Prompt de Comando (tecla Windows + R → digite cmd → Enter)

    Digite um comando de cada vez (aperte Enter depois de cada):

cmd

cd C:\agentes

cmd

docker compose up -d

    Vai aparecer uns textos baixando. AGUARDE.

    Quando voltar a piscar o cursor, digite:

cmd

docker compose ps

VOCÊ DEVE VER 6 LINHAS COM "Up" NA COLUNA STATUS
PASSO 6: CARREGAR OS MODELOS DENTRO DE CADA CONTAINER

ATENÇÃO: Este passo é necessário porque o Ollama não carrega GGUF automaticamente.

Para CADA modelo, você precisa fazer isso:
Para KAIYA:
cmd

docker exec -it kaiya ollama create kaiya --model /root/.ollama/models/KAIYA_modelo_q4_0.gguf

Para NYRA:
cmd

docker exec -it nyra ollama create nyra --model /root/.ollama/models/NYRA_Q4_KM.gguf

Para LUMINA:
cmd

docker exec -it lumina ollama create lumina --model /root/.ollama/models/Qwen_LUMINA_Q4_K_M.gguf

Para WELLINGTON:
cmd

docker exec -it wellington ollama create wellington --model /root/.ollama/models/Qwen_WELLINGTON_Final_Q4_K_M.gguf

Para EVA:
cmd

docker exec -it eva ollama create eva --model /root/.ollama/models/tinyllama_base_EVA_q4_0.gguf

Para YUNA:
cmd

docker exec -it yuna ollama create yuna --model /root/.ollama/models/tinyllama_base_Yuna_Q4_K_M.gguf

Cada comando demora alguns segundos para processar

⚠️ PROBLEMAS COMUNS E SOLUÇÕES REAIS
"nvidia driver not found"

    Driver da NVIDIA desatualizado

    Solução: Baixar driver novo no site da NVIDIA

"port is already allocated"

    Algum programa já está usando a porta

    Solução: Mude as portas no compose.yaml (ex: 11501, 11502...)

"out of memory"

    GPU não aguentou todos modelos

    Solução: Pare um dos pesados (lumina ou wellington) com docker stop lumina

Container não sobe

    Verifique se os caminhos das pastas estão corretos

    O Windows usa C:\caminho mas no YAML tem que ser C:\caminho
